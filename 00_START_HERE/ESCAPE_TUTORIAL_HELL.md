## üìö How to Learn (The Right Way)

### The Just-In-Time Learning Method

```
1. Start building
2. Get stuck
3. Learn ONLY what you need to get unstuck
4. Continue building
5. Repeat
```

### Example Flow

‚ùå **WRONG:**
1. Watch 20-hour React course
2. Watch TypeScript course
3. Watch Next.js course
4. Watch database course
5. (Never build anything)

‚úÖ **RIGHT:**
1. Start building todo app
2. Error: "Can't connect to database"
3. Google: "How to connect PostgreSQL in Next.js"
4. Read the first good result
5. Fix the issue
6. Continue building

### Documentation > Tutorials

**When stuck, check in this order:**

1. **Official docs** (nextjs.org, react.dev, etc.)
2. **GitHub issues** (someone had your problem)
3. **Stack Overflow** (recent answers only)
4. **ChatGPT/Claude** (paste your error)
5. **YouTube** (last resort, watch at 2x speed)

### Maximum Tutorial Time

- **Short videos** (< 10 min): Okay occasionally
- **Long courses** (> 2 hours): BANNED for 90 days
- **"Complete X in Y hours"**: Delete immediately

## üéØ Anti-Tutorial-Hell Rules

### The 10 Commandments

1. **No course without a project plan**
   - Before starting ANY tutorial: Plan what you'll build with it
   - If you can't think of a project, skip the tutorial

2. **Build alongside, not after**
   - Don't wait to finish the course
   - Build your version while watching
   - Diverge from the tutorial on purpose

3. **The 2-Hour Rule**
   - Max 2 hours of learning per day
   - Min 4 hours of building per day
   - No exceptions

4. **Deploy or delete**
   - Every project must be deployed
   - If not deployed in 1 week, delete it
   - Harsh but necessary

5. **No "learning projects"**
   - All projects must solve a real problem
   - Or be portfolio-worthy
   - No throwaway code

6. **Public accountability**
   - Share your work daily
   - Tweet progress with #100DaysOfCode
   - GitHub commits must be green

7. **One stack until mastery**
   - No "learning" new frameworks
   - Master one stack completely
   - Then (maybe) explore others

8. **Tutorials must be outdated-proof**
   - Official docs > everything
   - If tutorial is > 1 year old, skip it
   - Modern tools change fast

9. **Track builds, not courses**
   - Count projects deployed
   - Count features shipped
   - Don't count courses completed

10. **Done > Perfect**
    - Ship buggy code
    - Fix in production
    - Iterate based on real usage

## üö® When You Get Stuck

### The 30-Minute Rule

When you hit a problem:

**0-15 min:** Try to solve it yourself
- Read error message carefully
- Check your code for typos
- Use console.log / debugger
- Re-read relevant docs

**15-30 min:** Search for answers
- Google the exact error
- Check Stack Overflow
- Read GitHub issues
- Ask ChatGPT

**30+ min:** Ask for help
- Post in Discord/Slack
- Tweet with #DevCommunity
- Ask on Stack Overflow
- Pay for mentor session (Codementor)

**NEVER** spend 4+ hours stuck on one problem. Ask for help.
